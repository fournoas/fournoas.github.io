十款 Windows 免费音频播放器评测

如今的音乐软件已经是网络播放器的天下，但是对于热衷本地囤货的赛博老饕而言，一款本地音乐播放器仍然是必不可少的装备。

这里选择八款 Windows 平台下的免费音乐播放器软件进行评测。用于测试的音频文件总计 576 GB，包含 12,858 个文件，2,550 个文件夹，其中有部分音频在压缩包中。

foobar2000

foobar2000 是 Windows 平台上的老牌音乐播放器。支持第三方插件，可定制程度高。通过插件可以支持 APE 格式、歌词和压缩包。

载入测试音频文件夹大约耗时 2 分钟，共识别出 26755 个音轨。软件载入音轨后内存占用 40 MB 左右。播放时内存占用 50 MB 左右，CPU 占用 1% 以下。由于软件默认不载入 *.cue 文件，需要在设置 File > Preferences > Shell Integration > Exclude file types 中去除 *.CUE 文件类型。除默认安装的插件外，还安装了第三方插件：Album list panel、Columns UI、Monkey’s Audio Decoder 和 OpenLyrics。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✕
  • 压缩包：✓
  • 歌词：✓（需要安装 OpenLyrics 插件）
  • 音乐管理：✓
点评

原生的 foobar2000 没有花哨的界面，对有些人来说是优点，对有些人来说是缺陷，见仁见智。此外，官方软件也不提供中文界面或者其他汉化手段。网上虽有爱好者发布的界面美化版或汉化版，不过出于安全性考虑，不建议安装此类来历不明的版本。原生软件几乎无法开箱即用，需要手动安装插件和配置后方能使用顺手，对于初级用户存在较高的使用门槛。顺便一提，foobar2000 是此次评测中唯一一款同时支持歌词显示和加载压缩包音频文件的播放器。

对于有一定的软件使用经验、乐于折腾，且喜欢极致纯粹的音乐播放器的用户，推荐使用此软件。

  • 官方网站：https://www.foobar2000.org

DeaDBeeF

一款支持跨平台的开源音频播放器。GUI 界面基于 GTK 开发，用户界面接近 foobar2000。软件在 Windows 平台上启动比较慢，大约耗时 4 秒左右。

载入测试音频文件夹大约耗时 2 分 10 秒，共识别出 25,536 个音轨。软件载入音轨后内存占用 50 MB 左右。播放时内存占用 53 MB 左右，CPU 占用 1% 以下。除默认安装的插件外，还安装了第三方插件 RAR, 7z and Gzip archive reader。遗憾的是，DeaDBeeF 在 Windows 平台下没有可用的歌词插件。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✓
  • 歌词：✕
  • 音乐管理：✕
点评

比 foobar2000 还要纯粹的音乐播放器。没有歌词，没有专辑封面，也没有音乐管理功能，好在支持的音频格式算是比较全面。虽然支持插件，但是在 Windows 上几乎没有任何可用的第三方插件。虽然启动稍慢了些，但占用的系统资源很少，和 foobar2000 不相上下。到也对得起软件的 Slogan: The Ultimate Music Player。

如果你对开源软件有偏爱，或者只想工作时在后台播放一点背景音乐，而不需要额外的功能，那么可以尝试一下。

  • 官方网站：https://deadbeef.sourceforge.io

Audacious

又是一款支持跨平台的开源音频播放器。支持 GTK 和 Qt 两种界面，另外还可以切换到 Winamp 古典风格。

载入测试音频文件夹大约耗时 50 秒，共识别出 21,593 个音轨。软件载入音轨后内存占用 30 MB 左右。播放时内存占用 30 MB 左右，CPU 占用 1% 以下。当 Audacious 运行在 Qt 模式下时，可以启用歌词插件，只是不怎么好用。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
  • 音乐管理：✕
点评

界面秉持了开源软件一贯的朴实无华。相对 DeaDBeeF，至少能显示专辑封面，可惜外挂歌词的插件不好使，歌词也没有时间轴信息。最让人无法接受的是，播放器不支持加载压缩包内的音乐文件。

相当鸡肋的播放器，实在找不到推荐的理由。如果你想体验一把开源版的 Winamp，或许可以尝试一番。

  • 官方网站：https://audacious-media-player.org

MusicBee

MusicBee 是 Windows 平台专有软件。软件采用 BASS 音频库，GUI 使用 VB.Net 开发。播放器可以切换到简洁模式和迷你模式。

载入测试音频文件夹大约耗时 15 秒，共识别出 23,217 个音轨。软件载入音轨后内存占用 55 MB 左右。播放时内存占用 58 MB 左右，CPU 占用 1% 以下。曲库扫描速度非常快。自带的歌词插件不太好用，第三方的歌词插件暂时只找到一个：MusicBee-DesktopLyrics，但似乎无法正常工作。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
  • 音乐管理：✓
点评

MusicBee 是 Windows 上比较流行的音乐播放器。界面美观，支持换肤，具有一定的音乐管理功能，还支持网络电台和视频播客。可惜也不支持加载压缩包内的音乐文件。另外不知为何，我一直没能让软件的歌词插件正常工作起来。

对于不想折腾且对高级功能没有需求的用户，无脑选它就是了。

  • 官方网站：https://www.getmusicbee.com

AIMP

源自俄罗斯的一款 Windows 平台专有软件。软件支持换肤，并且可以切换界面风格。除了网络播客外，软件还支持 WebDAV 和各种网盘。

载入测试音频文件夹大约耗时 2 分钟，共识别出 23,391 个音轨。软件载入音轨后内存占用 58 MB 左右。播放时内存占用 100 MB 左右，CPU 占用 1% 以下。在操作软件时，内存占用波动比较大，仅供参考。不过当软件最小化到托盘后，内存占用瞬间降低到 6 MB 左右。

  • 内存占用：中
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
  • 音乐管理：✓
点评

个人认为 AIMP 无论是颜值还是功能都要比 MusicBee 高出一个档次。最大的特色是支持 WebDAV 和各种网盘。虽然自带的歌词功能不太好用，但是可以安装外挂的可道歌词来提升用户体验。唯一的缺憾就是它也不支持加载压缩包内的音乐文件。

应该是 Windows 下第二好用的音乐播放器了。

  • 官方网站：https://www.aimp.ru

MediaMonkey

相当浮夸的音乐播放器。首次启动软件后不做任何操作，内存占用直接突破 130 MB。MediaMonkey 支持 Dropbox、Google Drive、OneDrive 云盘，还支持 DLNA 服务。

载入测试音频文件夹报错，大部分音轨都未识别出来。在操作过程中还出现错误要求重启软件。软件载入音轨后内存占用 227 MB 左右。播放时内存占用 253 MB 左右，CPU 占用 2% 以上。

  • 内存占用：高
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
点评

丑陋的橙红色的界面开了让人血压升高。扫描音频文件的操作会被错误打断，这种产品设计非常不合理。系统资源占用太过浮夸，对旧设备来说简直致命。软件代码也不健壮，第一次碰到音乐播放器运行崩溃的场面。更可恶的是，这款软件卸载后还会在注册表中残留一堆垃圾数据，并且不会复原文件默认打开方式。

软件所有的优点都无法弥补其缺陷，完全就是一坨垃圾。

  • 官方网站：https://www.mediamonkey.com

Clementine

Clementine 是一款多平台音乐播放器。源自另一个开源播放器 Amarok。和大多数的开源跨平台程序一样，界面不怎么美观。支持网络广播和大多数主流网盘，其支持的网盘数量远胜于其他播放器。该播放器还有一个分叉项目——Strawberry Music Player，其界面和功能与 Clementine 相似，侧重极致的音质体验，但对网络功能支持不佳。

载入测试音频文件夹耗时 5 分钟以上，共识别出 21,161 个音轨。软件载入音轨后内存占用 66 MB 左右。播放时内存占用 120 MB 左右，CPU 占用 1% 以下。内置歌词支持，不需要安装插件，不过歌词没有时间轴信息。Clementine 似乎不支持 UTF-8 编码的 ID3。

  • 内存占用：中
  • 插件扩展：✕
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
点评

界面美观度勉强达到可接受的水平。依然是不支持加载压缩包。播放器唯一的优势是支持数量众多的网络云盘。

如果你有大量音乐资源存放在各种网盘中，推荐使用这款播放器。

  • 官方网站：https://www.clementine-player.org

Exaile

一款使用 Python 开发的跨平台播放器。用户界面使用 GTK+，音频解码基于 GStreamer。

载入测试音频文件夹耗时 45 秒左右，共识别出 2,358 个音轨。软件载入音轨后内存占用 40 MB 左右。播放时内存占用 45 MB 左右，CPU 占用 1% 以下。由于 Exaile 支持的音频格式比较少，且无法加载压缩包，所以识别出的音轨数量很少。音频文件 ID3 中的中文字符会显示为乱码。插件使用 Python 开发，非常易于扩展。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✕
  • 压缩包：✕
  • 歌词：✓
  • 音乐管理：✓
点评

用户界面十分清新。可惜默认没有中文界面，对中文支持也有问题，默认支持的格式又少得可怜。是一款需要自己折腾的播放器。另外，Windows 版本还存在一个致命缺陷，软件在构建时没有打包所需的所有 Python 库，导致包括歌词插件在内的部分插件不能正常工作。

非常可惜，Windows 版本终究只是个鸡肋罢了。

BTW: 还有一款类似的基于 Python + GStreamer 后端的音乐播放器 Quod Libet ，缺点也是类似。

  • 官方网站：https://exaile.org

VUPlayer

一款非常朴（jian）素（lou）的开源音乐播放器。支持 APE 和 FLAC 格式，界面类似 Foobar2000 ，不支持歌词插件。

载入测试音频文件夹耗时 10 分钟以上，共识别出 3004 个音轨。软件载入音轨后内存占用 55 MB 左右。播放时内存占用 65 MB 左右，CPU 占用 1% 以下。

  • 内存占用：低
  • 插件扩展：✕
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✕
  • 音乐管理：✓
点评

相当于 Foobar2000 的平替。优点是不需要配置，开箱即用。

官方网站：http://www.vuplayer.com

Musikcube

一款基于命令行的开源音乐播放器，同时也可以作为流媒体服务器运行。此外， Musikcube 还有对应的 Android 客户端 musikdroid 。

载入测试音频文件夹耗时 20 秒以内，共识别出 3000 个音轨。软件载入音轨后内存占用 15 MB 左右。播放时内存占用 15 MB 左右，CPU 占用 1% 以下。

  • 内存占用：低
  • 插件扩展：✓
  • 开箱即用：✓
  • 压缩包：✕
  • 歌词：✓
  • 音乐管理：✓
点评

不太适合当成桌面播放器，可以当成一款带播放器功能的流媒体服务器软件。另外，软件不支持 CUE 播放列表，也不支持在线歌词搜索。

官方网站：https://musikcube.com

总结

非常可惜，几款跨平台开源播放器在 Windows 上的表现都不佳，可以说一个能打的都没有。foobar2000 无愧于 Windows 老牌播放器的英名，二十年后尚能一战。MusicBee 和 AIMP 在 foobar2000 面前，除了用户界面更加现代一点，似乎并没有什么优势。

如果你曾经是 foobar2000 的用户，那完全没有理由更换播放器；如果你是一个新手用户，则可以在 MusicBee 或 AIMP 中选择一款界面中意的播放器；如果你对开源软件情有独钟，且只需要一款简单纯粹的播放器，那么可以考虑 DeaDBeeF。
