Windows 下使用 Neovim

作者 Fournoas 于 2023年7月20日 发布在 
  • Using

虽然在 Windows 下的 VSCode 用得挺好，但还是想折腾一下 Neovim 试试。Neovim 安装完毕后只是一个简陋的编辑器，需要自己配置和安装各种插件才能进行日常使用，幸好已经有很多现成的开源配置项目，像 AstroNvim 和 LazyVim 等。经过试用，发现 AstroNvim 在 Windows 上有一些为解决的 BUG ，比如一直报「 E65: Illegal back reference 」错误。此外， AstroNvim 的代码也比较混乱。因此，最后选择使用 LazyVim 的配置。

安装 LazyVim

本文所有软件和工具安装都使用 Scoop 。安装 Scoop 可以参考之前的文章《Windows 包管理工具》。

为了显示 Neovim 界面中的图标，需要安装一款 Nerd Font 字体，这里选择 UbuntuMono Nerd Font 字体。首先，打开系统的「字体设置」页面，将下载的字体文件拖进虚线框中进行安装：

接下来，打开 Windows 终端的「设置」页面，在「配置文件 - 默认值 - 其他设置 - 外观」的「字体」下拉框中，选中「 UbuntuMono Nerd Font 」字体并保存：

由于众所周知的原因，安装前需要配置科学上网（根据自己的网络环境修改代理服务器地址）：

$Env:http_proxy='http://127.0.0.1:7890'
$Env:https_proxy='http://127.0.0.1:7890'
scoop config proxy 127.0.0.1:7890

安装必要组件：

scoop install windows-terminal neovim git lazygit gcc ripgrep fd unzip tree-sitter

虽然 LazyVim 官方文档要求 Windows 下安装 alacritty 或 wezterm 终端，但其实 Windows Terminal 已经足够了。

为 Git 访问 GitHub 设置代理：

git config --global http.https://github.com.proxy socks5://127.0.0.1:7890

安装 LazyVim 配置：

git clone https://github.com/LazyVim/starter $env:LOCALAPPDATA\nvim --depth=1

初始运行

在命令行中使用 nvim 命令就可以启动 LazyVim 了。程序第一次运行时会自动下载和安装插件。如果遇到网络问题而导致安装失败也没关系，按下 R （大写）重新安装直至成功为止。所有插件安装成功后， nvim-treesitter 还会继续安装组件，等所有插件和组件安装完毕后，按下 q 退出安装界面。

意外的是，在 nvim-treesitter 安装 regex 组件时遇到了奇怪的错误：

这是 Git 报的错，原因是 tree-sitter-regex 仓库 checkout 了一个无效的 commit ID 。这应该是 nvim-treesitter 的一个 BUG 。

打开 %LOCALAPPDATA%\nvim-data\lazy\nvim-treesitter\lockfile.json 文件，定位到第 438 行：

   "regex": {
     "revision": "17a3293714312c691ef14217f60593a3d093381c"
   },

将 revision 的值改为 2354482d7e2e8f8ff33c1ef6c8aa5690410fbc96 并保存，运行 VIM 命令 :TSUpdate all 强制更新。

安装 LSP

运行 VIM 命令 :Mason 打开 mason.nvim 配置界面，按 2 进入 LSP 配置页面。如果需要安装 Python 的 LSP ，按下 ctrl + f ，输入 python 然后回车，可以看到所有的 Python LSP ，选中其中的 pyright ，按下 i 进行安装（ BTW ：按 X 卸载）。
