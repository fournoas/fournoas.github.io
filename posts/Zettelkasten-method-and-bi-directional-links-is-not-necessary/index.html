不要迷信卢曼卡片盒方法和双向链笔记

作者 Fournoas 于 2021年3月11日 发布在
  • Using

卡片盒笔记方法是一种通过扁平化的链接来组织笔记的方法。事实上，卡片盒笔记也并非完全扁平化的，也存在结构化的索引表和枢纽笔记。

双向链就是一种 trackback。简而言之，即A通过链接连接到B，那么B将自动生成对A的反向连接关系，当用户访问B时，也可以通过反向连接关系来回溯到A，从而让笔记系统形成一个可遍历的无向图。

其实，相似的系统早就存在，并且大家已经习以为常，那就是维基百科。事实上，浏览维基百科的体验并不是很好——我们往往沉浸于逐条访问百科词条中的无关链接，等回过神来已经几个小时过去了。

从上面的描述来看，这种笔记系统并非适合所有人和所有场景。首先，卡片盒方法中的笔记是一种碎片化的信息，需要通过连接才能组织成一个成体系的内容；其次，双向链是一种发散的非收敛的阅读方式，很容易就让检索方向偏离最初的目标。这种笔记组织方式完全不符合本人的思维模式。

就我个人而言，笔记的内容来源大多来自网络采集而非个人记录，所以对我来说，信息检索需求要远远高于记录需求。我需要的是一种能快速收敛的检索系统，而不是漫无目的地爬虫式的信息漫游。那些只言片语的零碎想法只会停留在我脑中萦绕，只有当碎片信息形成系统性的观点后，我才会将其记录下来。即便有实时记录的需求，我也习惯按照主题分类，将相同主题的信息汇总在同一个文档中，而不是分散保存。因为信息采用电子化保存，我可以很方便地添加或修改原有的内容，而不需要通过新增卡片的形式来拓展内容。通过修改日志也可以方便地对内容进行回滚。

我理想中的笔记系统更接近内容管理系统，应该具备如下的特性：

  • 支持层级的分类系统
    一篇文章可以归类至多个分类下。按分类检索时，可以仅显示当前分类下的文章或同时显示子分类下的文章。
    这么做的好处是，标签系统可以直接用分类系统替代。

  • 全文搜索
    这个是必须的。

  • 完善的富文本编辑器
    至少支持 Markdown 和 HTML。从浏览器复制黏贴的格式不要错乱，支持图片自动上传。

  • 按分类进行交集、并集、差集检索
    很多CMS系统都不支持这个功能，但是对笔记系统来说还是挺需要的。

  • 修改日志
    保证文章可以回滚到任意历史版本。

  • 提供 Web 远程访问
    本地客户端完全没有必要，只要提供HTTP服务，并且网页适配移动端就行了。

  • 方便地实现数据备份和迁移
    数据备份和迁移应该作为软件的内置功能而不需要手动操作。
