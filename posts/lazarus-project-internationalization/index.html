Lazarus 项目支持多语言国际化

作者 Fournoas 于 2023年10月7日 发布在 
  • Coding

Lazarus 是一个与 Delphi 兼容的跨平台 RAD 集成开发环境。和 Delphi 一样， Lazarus 可以拖放组件，快速开发 GUI 应用程序。它使用 Free Pascal 作为后端编译器。当前的 Lazarus 版本是 2.2.6 ，内置的 Free Pascal 编译器版本是 3.2.2 。

修复 Free Pascal 中 gettext 包的缺陷

Free Pascal 包含 gettext 包，可以用来实现软件国际化。但是当前版本 gettext 中的 GetLanguageIDs 过程在 Windows 系统下存在缺陷：

在简体中文的 Windows 环境中，该过程输出的 Lang 值是 ch_CH ，而不是期望的 zh_CN 。虽然在最新的 FPCSource 源代码中已经修复了这个问题，但事实上修复方案是有问题的：

可以看到，修复方法仅仅是为了中文环境做了特殊处理。然而引发这个缺陷的主要原因是代码中的 GetLocaleInfo 调用传递了错误的参数。

Windows API GetLocaleInfoA 的原型如下：

C++int GetLocaleInfoA(
  [in]            LCID   Locale,
  [in]            LCTYPE LCType,
  [out, optional] LPSTR  lpLCData,
  [in]            int    cchData
);

在 GetLanguageIDs 中调用该 API 时，第二个参数 LCType 分别传递的是 LOCALE_SABBREVLANGNAME 和 LOCALE_SABBREVCTRYNAME 。事实上在 Windows SDK 中已经明确标注了这两个常量已经过时，需要用 LOCALE_SISO639LANGNAME 和 LOCALE_SISO3166CTRYNAME 替代。

正确的修复方法是将 GetLanguageIDs 代码替换为：

PASCAL{$ifdef windows}
procedure GetLanguageIDs(var Lang, FallbackLang:AnsiString );
var
  Buffer: array[1..4] of {$ifdef Wince}WideChar{$else}AnsiChar{$endif};
  Country: AnsiString;
  UserLCID: LCID;
begin
  Lang := '';
  FallbackLang:='';
  UserLCID := GetUserDefaultLCID;
  if GetLocaleInfo(UserLCID, LOCALE_SISO639LANGNAME, @Buffer[1], 4)<>0 then begin
    FallbackLang := lowercase(copy(Buffer,1,2));
  end;
  if GetLocaleInfo(UserLCID, LOCALE_SISO3166CTRYNAME, @Buffer[1], 4)<>0 then begin
    Country := copy(Buffer,1,2);
    Lang := FallbackLang+'_'+Country;
  end;
end;

可以在当前项目下新建 gettext.pas 单元，并将修复的代码写入保存，用于覆盖编译器自带的 gettext 包。

在 Lazarus 项目中使用 gettext 本地化

新建一个 Lazarus 项目，默认窗口名为 Form1 ，默认单元名为 Unit1 。假设要本地化的字符串是窗口的标题。

在主窗口单元中定义 resourcestring 节，在该节中声明所有需要翻译的资源字符串。并在 uses 节中引用 gettext 模块：

为主窗口类添加 Localize 方法，并在主窗口的 FormCreate 事件中调用：

PASCALprocedure TForm1.Localize();
begin
  TranslateResourceStrings('./lang/%s.mo');
  Caption := Form1Caption;
end;

procedure TForm1.FormCreate(Sender: TObject);
begin
  Localize;
end;

TranslateResourceStrings 会调用 GetLanguageIDs 获取本地语言代码，并将 './lang/%s.mo' 中的 %s 替换成当前的语言代码。如果是简体中文环境，那么将加载 ./lang/zh_CN.mo 文件。

项目编译后，会在当前项目路径的 lib\x86_64-win64 目录下生成一个 unit1.rsj 文件。用 Free Pascal 自带的 rstconv 命令生成 POT 文件：

CMDrstconv -i unit1.rsj -o unit1.pot

rstconv 命令在 Larazus 安装路径下的 fpc\3.2.2\bin\x86_64-win64 目录内。

然后通过 Poedit 翻译并生成 MO 文件。将文件命名为 zh_CN.mo ， 保存在项目 lang 目录下。在 Lazarus IDE 中按下 F9 ，就能看到窗体的标题显示为中文了。
