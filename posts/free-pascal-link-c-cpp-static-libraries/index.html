Free Pascal 链接 C/C++ 静态库

作者 Fournoas 写于 2025年4月2日 发布在 
  • Coding

原项目是个 C 语言编写的命令行工具，打算用 Lazarus 给它加上 GUI。另外为了便于分发，不打算使用 DLL 动态链接或者 GUI Shell 的方式，而是编译成单一的 EXE 可执行文件。只要将原项目的核心业务封装成一个静态库，便可以分别供 CLI 和 GUI 调用。

首先将 C 代码编译为静态库，然后在 Lazarus > 项目选项 > 编译器选项 > 路径 > 库 (-Fl) 中输入静态库存放的路径。此处以静态库导出 MyAdd() 函数为例，演示在 Free Pascal 中如何调用它。

编译后生成的静态库文件名为 libmylib.a。

以下是一些踩坑记录：

  1. 静态库的扩展名必须是 .a 。如果是其他扩展名，fpc 会将其视作 DLL 动态链接库，导致链接失败；
  2. fpc 仅支持 gcc 编译器生成的静态库；
  3. 如果是 MSVC 编译的静态库，需要使用 ranlib 命令（包含于 WinLibs）转格式，然后重命名为 libmylib.a；
  4. fpc 不支持 clang 编译器生成的静态库，似乎是 COFF 格式有问题。
